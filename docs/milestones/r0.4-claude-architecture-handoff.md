## Approved Scope

User-visible goal:

Replace exposed continuous Pulse with a sparse one-shot Resonance accent while
repairing verified R0.3 remainder issues reported through human QA.

Must-haves:

- Remove Pulse from the visible continuous mixer, presets, default mix, and
  intention nudges.
- Stop building the continuous Pulse graph.
- Preserve compatibility with old mixes containing `pulse` or `tape`.
- Ignore legacy `pulse` as an effective current layer.
- Continue mapping legacy `tape` to Analog.
- Add a safe one-shot procedural Resonance API with no recurring scheduler.
- Preserve the R0.2/R0.3 transition mutex, fade safety, and audible-state rules.
- Start the visual timer immediately after accepted Begin/Resume intent rather
  than after the 2.4-second audio fade.
- Verify and repair active-draft clearing, route identity, timer, and
  `createdAt` behavior only where tests reproduce a defect.
- Add a quiet empty-journal state to History while preserving existing
  expand/collapse, export, delete-one, and delete-all behavior.
- Null `_dropletBuf` and `_analogCrackleBuf` in `dispose()`.
- Move the `tape` compatibility alias out of `buildAnalog`.
- Update `docs/AUDIO_ARCHITECTURE.md` for the modular R0.3 runtime and R0.4
  event-based Resonance.
- Preserve slider zero as true silence.
- Record human listening QA before audio acceptance.

Human-reported audio polish candidates:

- Reduce Rain dominance at very high values.
- Add subtle Pad stereo/depth only if low-risk.
- Make Analog crackle slightly more perceptible while remaining sparse.

These are listening-calibrated changes, not architecture assumptions.

Explicit non-goals:

- backend, auth, accounts, cloud sync, or AI;
- imported audio files;
- frequency-healing or 432 Hz claims;
- scene architecture;
- binaural, HRTF, spatial 3D, or recurring Resonance scheduling;
- visual redesign or unrelated UI rewrite;
- full resume of completed sessions;
- breaking localStorage records or deleting old session history;
- new runtime dependencies;
- opportunistic layer-builder architecture refactor.

Human-approved constraints:

- R0.3.1 documentation repair and R0.4 implementation occur in this milestone.
- Do not merge until human browser and listening QA are complete.
- Global mute is optional and must be deferred if it expands audio
  architecture.
- A manual Resonance/Strike control is optional and must not clutter the UI.

   engine behavior.

13. Add a draft-persistence guard/ref that disables all autosave paths before
    End & Save or Discard clears the draft.
14. Define behavior for mode/mood identity changes. Preferred approach:
    initialize/reset session state from a route identity boundary rather than
    allowing stale state to cross query changes.
15. Preserve fresh zero timer and fresh `createdAt`; only change these when a

## Risks And Rollback

Technical risks:

- A legacy alias may accidentally recreate an effective Pulse value.
- A Resonance source may leak nodes or survive pause/dispose.
- Timer and draft state may diverge during audio failure or transition.
- Clearing a draft may race with autosave.
- Route changes may carry stale session state.
- Rain/Pad/Analog tuning can change audible balance without automated proof.

UX/audio/data risks:

- Resonance transient may be harsh or too loud on headphones.
- End resonance may be inaudible or feel delayed under fade-out.
- Immediate timer start must not falsely claim that audio succeeded.
- History changes must not alter saved records.
- Legacy saved sessions must remain readable and exportable.

Rollback path:

- Revert Resonance files/API and restore previous graph calls/constants.
- Restore Pulse entries only if rollback is required; do not migrate/delete
  stored data.
- Revert timer predicate and draft guard independently from audio work.
- Revert listening-tuning commit separately.
- Revert documentation last so it continues matching implementation.
- No dependency removal is expected for R0.4.
